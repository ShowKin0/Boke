# 个人博客网站设计文档

## 概述

一个粉白蓝配色的单页个人博客网站 + 后台管理系统。包含首页、文章、动态、探索四个区域，支持平滑滚动导航，毛玻璃效果，丰富的交互动画。所有内容类型（文章、动态、探索）均支持图片、音频、视频多媒体嵌入。后台可管理文章、动态、探索内容、音乐及主题配色。

## 设计原则

### 🎨 配色统一
- 全站严格限定在粉白蓝色系范围内，所有元素（导航栏、卡片、按钮、播放器、时钟、标签、链接等）均从统一的 CSS 变量取值
- 后台自定义配色实时同步到所有组件，无例外
- 文字色只用深灰 `#2d2d2d` 和灰色 `#888888` 两级，保持清晰层级
- 强调色（粉色 `#ffb0c0`）用于交互元素（悬停、选中、高亮）；辅助色（蓝色 `#87ceeb`）用于信息元素（标签、链接、图标）
- 毛玻璃效果统一使用 `rgba(255,255,255,0.6)` 背景 + `backdrop-filter: blur(20px)`，保持一致的通透质感

### 🔄 交互不重叠
- **导航栏**：顶部水平 ↔ 左侧垂直 切换时有平滑过渡（0.5s），过渡期间禁用其他交互，避免动画冲突
- **滚动触发动画**：使用 IntersectionObserver 的 `threshold` 精确控制触发时机，与导航栏高亮逻辑解耦，互不干扰
- **卡片交互**：悬浮效果（hover）与点击展开互不冲突，点击时不触发悬浮态
- **音乐播放器**：全局唯一音频实例，切换歌曲时自动停止当前播放，避免多音频重叠
- **视频/音频播放**：同一时间只能播放一个媒体实例，播放新媒体时自动暂停当前播放的媒体（含音乐播放器）
- **后台管理**：表单提交时禁用重复点击，CRUD 操作有加载状态，防止重复提交
- **滚动与动画**：滚动触发的入场动画只执行一次，不重复触发

## 技术选型

- **前端**：纯 HTML + CSS + JavaScript（`index.html`）
- **后端**：Node.js + Express
- **数据存储**：JSON 文件（`data/` 目录）
- **文件存储**：`public/uploads/` 目录（图片、音频、视频文件）
- **认证**：简单 Session/Cookie 密码认证
- **CSS 变量**管理主题色
- **IntersectionObserver** 实现滚动动画
- **CSS `backdrop-filter`** 实现毛玻璃效果

## 模块详情

### 首页

**数字时钟：**
- 大号数字显示（时:分:秒）
- 立体边框设计（3D 框架效果）
- 时钟阴影方向根据当前时间变化（模拟太阳位置，0时→左，6时→下，12时→右，18时→上）
- 下方显示当前日期

**最新文章 × 3：**
- 三篇最新文章的毛玻璃卡片预览
- 显示标题、摘要、日期、封面图及多媒体标记（🎵🎬 等）
- 点击可跳转到文章区对应位置

**最新动态 × 2-3：**
- 最近 2-3 条动态预览
- 简化版社交卡片样式，展示图片/音视频标记

**音乐播放器：**
- 混搭方案：自制简约播放器 + 支持外链歌单
- 播放/暂停、切歌、进度条、音量控制
- 歌曲列表可展开
- 预设几首轻音乐
- 支持通过后台添加外链歌单（网易云/QQ音乐）

### 文章区

- 杂志式布局
- 首篇文章大图展示（占两列宽）
- 其余文章以瀑布流卡片排列
- 每张卡片含：封面图、标题、摘要、日期、标签
- **多媒体支持**：文章内容支持插入图片、音频播放器、视频播放器（HTML 富内容）
- 卡片右下角显示多媒体类型标记（如包含视频则显示 🎬 标记）
- 卡片带毛玻璃效果
- 悬浮动画效果

### 动态区

- 社交式卡片风格
- 每条动态显示：头像、昵称、内容文字、时间
- **多媒体支持**：
  - 图片：多图网格展示（1-9 张）
  - 音频：内嵌音频播放器
  - 视频：内嵌视频播放器
  - 混合：图文+音频/视频 组合
- 多媒体内容以九宫格/卡片形式展示在文字下方
- 类似朋友圈/微博的卡片样式
- 按时间倒序排列

### 探索区

- 标签筛选模式
- 所有项目混合排列

- 每个项目卡片：图标、标题、描述、链接
- **多媒体支持**：每个探索项可附带封面图、预览视频/音频
- 视频类探索项自动展示预览缩略图，点击可播放

## 动画效果

1. **导航栏动画** — 顶部↔左侧切换过渡（0.5s ease-in-out）
2. **滚动触发动画** — 元素进入视口时从下方滑入并淡出（IntersectionObserver，透明度 0→1，translateY 40px→0）
3. **卡片悬浮效果** — 悬浮时上浮 -8px + 阴影增强
4. **时钟阴影动画** — 阴影方向和长度随时间平滑变化
5. **页面区域指示器** — 滚动时导航栏高亮当前区域（底部/右侧滑块指示）
6. **播放器动画** — 播放时旋转唱片/封面动画
7. **区域切换动画** — 进入新区域时的淡入效果
8. **多媒体灯箱效果** — 点击图片/视频可放大预览

## 多媒体支持详述

### 图片
- 支持的格式：jpg, png, gif, webp, svg
- 上传：后台拖拽或点击上传，自动缩略图
- 展示：自适应图片网格、灯箱放大预览、懒加载
- 适用：文章封面/插图、动态配图、探索项封面

### 音频
- 支持的格式：mp3, wav, ogg, aac
- 上传：后台文件上传
- 展示：内嵌音频播放器（自定义 UI，与音乐播放器风格统一）
- 适用：文章内嵌音频、动态语音/音乐分享、探索音频资源

### 视频
- 支持的格式：mp4, webm, mov
- 上传：后台文件上传（或嵌入外链如 B站/YouTube）
- 展示：内嵌视频播放器、封面缩略图预览、灯箱播放模式
- 适用：文章视频内容、动态视频分享、学习视频探索项

## 多媒体上传 API

```
POST /api/upload
Content-Type: multipart/form-data
参数: file (文件), type (image|audio|video)
返回: { "url": "/uploads/images/xxx.jpg", "filename": "xxx.jpg", "size": 12345 }
```

- 文件大小限制：图片 10MB，音频 50MB，视频 200MB
- 支持格式：图片(jpg/png/gif/webp)、音频(mp3/wav/ogg)、视频(mp4/webm/mov)
- 文件名自动生成 UUID + 原始扩展名
- 按类型存放到 `public/uploads/{images,audio,video}/` 目录

## 潜在问题与解决方案

### 1. JSON 文件并发写入冲突
- **问题**：多条请求同时写入同一 JSON 文件时，后一个写操作覆盖前一个，导致数据丢失
- **解决**：每个 JSON 文件加写入队列，使用 `read → modify → write` 原子操作模式；引入简单文件锁机制，同一时刻只允许一个写操作

### 2. 导航栏位置切换的临界闪烁
- **问题**：首页顶部 ↔ 左侧垂直的切换点（滚动到第一个 section 时）若用户快速滚动，可能出现反复切换的闪烁
- **解决**：使用 `getBoundingClientRect()` 精确计算 section 位置；切换逻辑加 100ms 防抖（debounce）；过渡动画期间锁定触发，避免连续切换

### 3. 全局音频互斥边界
- **问题**：用户可能在听背景音乐（音乐播放器）时想查看动态中的视频，粗暴的全局互斥会打断体验
- **解决**：分层互斥策略——
  - 音乐播放器（首页）与文章/动态音视频**不互斥**，可同时播放
  - 文章中的音视频与动态中的音视频**互斥**，播新的自动暂停旧的
  - 音量自动降低策略：播放页面音视频时，音乐播放器自动降低音量而非暂停

### 4. 首次加载空状态 / API 失败
- **问题**：页面加载时请求 5 个 API（文章/动态/探索/音乐/主题），任一失败会导致白屏或混乱布局
- **解决**：每个 API 调用独立 try-catch，失败时显示空状态占位卡片 + 重试提示；为每个数据模块设置默认空数据避免渲染报错

### 5. 大文件上传超时 / 限制
- **问题**：200MB 视频上传超出 Express 默认 100KB body 限制，且上传时间可能超时
- **解决**：multer 配置 `limits: { fileSize: 200 * 1024 * 1024 }`；上传接口关闭 bodyParser 限制；前端上传显示进度条（`XMLHttpRequest.upload.onprogress`）；超时 5 分钟提示

### 6. 移动端导航栏适配
- **问题**：左侧垂直导航栏在手机屏幕（< 768px）上会遮挡大量内容区域
- **解决**：移动端导航栏改为底部固定 tab 栏（水平排列）或可折叠汉堡菜单；滚动切换逻辑保留，但视觉适配屏幕宽度

### 7. 删除内容时媒体文件残留
- **问题**：删除文章/动态时，关联的图片/音频/视频文件未删除，长期积累浪费磁盘空间
- **解决**：删除 API 中遍历 mediaList，使用 `fs.unlink` 删除关联媒体文件；但需跳过可能被其他内容引用的文件（简单做法：个人博客场景不做引用计数，直接删除）

### 8. 后台配色修改后前台实时同步
- **问题**：后台修改配色后，前台页面如何即时生效
- **解决**：双重策略——
  - 前台每次加载页面时从 API 拉取最新配色
  - 后台保存配色后，若前台页面已打开，通过 `navigator.serviceWorker` 或 `window.postMessage`（同源 iframe 方式）通知刷新；或简单方案：前台每隔 60s 轮询 `/api/theme`

### 9. 文章内容 XSS 风险
- **问题**：文章内容使用 HTML 富文本，若直接 `innerHTML` 渲染存在 XSS 风险
- **解决**：因为是个人博客（唯一管理员），风险较低；但渲染前仍做基本 sanitize：过滤 `<script>` 标签、`onerror`/`onclick` 等事件属性

### 10. 数据文件首次初始化
- **问题**：首次启动时 `data/` 目录和 JSON 文件不存在，server.js 读取会报错
- **解决**：server.js 启动时自动检测 `data/` 目录和各 JSON 文件是否存在，不存在则创建目录并写入默认空数据数组 `[]`
